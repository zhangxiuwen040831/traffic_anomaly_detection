## 1. 项目背景与意义

随着网络攻击手段的不断演进，传统的基于规则的安全防护方法已经难以应对复杂的网络威胁。据统计，2025年全球网络攻击事件增长率超过40%，传统方法的检测率不足70%。

我们的项目旨在开发一种基于深度学习的无监督异常检测系统，能够自动识别偏离正常行为模式的网络流量，为网络安全提供新的保障手段。

## 2. 系统总体架构

我们的系统采用了五层架构设计：

1. **数据层**：负责数据采集、预处理和特征工程
2. **模型层**：使用Autoencoder系列模型进行无监督学习
3. **评估层**：进行多维度指标评估和阈值优化
4. **部署层**：实现模型打包和推理服务
5. **前端层**：提供用户友好的交互界面

这种分层设计使得系统具有良好的可扩展性和可维护性。

## 3. 核心算法设计

### 3.1 模型选择

我们测试了三种模型：
- **增强型MLP Autoencoder**：添加了批归一化和Dropout，提高了模型的泛化能力
- **VAE**：变分自编码器，能够更好地捕获数据分布
- **Hybrid AE**：结合多种评分组件

实验结果表明，增强型MLP Autoencoder在性能和计算效率之间取得了最佳平衡。

### 3.2 异常评分方法

我们提出了混合异常评分方法，结合了三种评分组件：
1. **重构误差**：衡量输入与重构输出的差异
2. **潜在空间距离**：计算样本在潜在空间中与正常样本分布的距离
3. **密度估计**：基于正常样本的密度分布评估异常程度

通过加权组合这三种评分方法，我们提高了检测的准确性。

### 3.3 阈值策略

我们实现了三种阈值选择策略：
- **F1最优阈值**：最大化F1分数
- **Youden指数**：平衡灵敏度和特异度
- **PR最优阈值**：最大化精确率和召回率的乘积

这些策略适应不同的应用场景需求。

## 4. 数据集与实验设置

我们使用了两个真实的网络安全数据集：
- **CIC-IDS2017**：包含多种网络攻击类型的真实数据集
- **UNSW-NB15**：网络安全研究常用数据集

评估指标包括：
- ROC-AUC：接收器操作特征曲线下面积
- PR-AUC：精确率-召回率曲线下面积
- F1 Score：精确率和召回率的调和平均
- Precision：精确率
- Recall：召回率
