# 网络流量异常检测系统 - 最终答辩报告

## 1. 项目概述

### 1.1 项目背景
随着网络攻击手段的不断演进，传统的基于规则的安全防护方法已难以应对复杂的网络威胁。异常检测作为一种基于机器学习的方法，能够识别偏离正常行为模式的网络流量，为网络安全提供新的保障手段。

### 1.2 项目目标
- 开发基于深度学习的网络流量异常检测系统
- 实现端到端的模型训练、评估和部署流程
- 构建用户友好的前端界面
- 验证系统在真实数据集上的性能

### 1.3 技术栈
- **后端**：Python, PyTorch, scikit-learn
- **前端**：Streamlit
- **部署**：Docker, REST API
- **数据处理**：Pandas, NumPy

## 2. 系统架构

### 2.1 总体架构
- **数据层**：数据采集、预处理、特征工程
- **模型层**：Autoencoder 系列模型，混合异常评分
- **评估层**：多维度指标评估，阈值优化
- **部署层**：模型打包，推理服务
- **前端层**：Streamlit 交互式界面

### 2.2 核心组件
- **数据预处理**：特征标准化、编码、缺失值处理
- **模型训练**：仅使用正常样本进行无监督学习
- **异常评分**：重构误差、潜在空间距离、密度估计的混合评分
- **阈值优化**：基于验证集的 f1_optimal、youden、pr_optimal 策略
- **推理服务**：批处理和单样本推理
- **前端界面**：数据上传、实时推理、结果可视化

## 3. 核心算法设计

### 3.1 模型选择
- **增强型 MLP Autoencoder**：添加批归一化和 Dropout，提高模型泛化能力
- **VAE**：变分自编码器，捕获数据分布
- **Hybrid AE**：结合多种评分组件

### 3.2 异常评分方法
- **重构误差**：衡量输入与重构输出的差异
- **潜在空间距离**：计算样本在潜在空间中与正常样本分布的距离
- **密度估计**：基于正常样本的密度分布评估异常程度
- **混合评分**：加权组合三种评分方法

### 3.3 阈值策略
- **F1 最优阈值**：最大化 F1 分数
- **Youden 指数**：平衡灵敏度和特异度
- **PR 最优阈值**：最大化精确率和召回率的乘积

## 4. 实验设置与结果

### 4.1 数据集
- **CIC-IDS2017**：包含多种网络攻击类型的真实数据集
- **UNSW-NB15**：网络安全研究常用数据集

### 4.2 评估指标
- **ROC-AUC**：接收器操作特征曲线下面积
- **PR-AUC**：精确率-召回率曲线下面积
- **F1 Score**：精确率和召回率的调和平均
- **Precision**：精确率
- **Recall**：召回率

### 4.3 主要结果

**CIC-IDS2017 + enhanced_mlp_ae**
- ROC-AUC = 0.9135
- PR-AUC = 0.8495
- F1 Score = 0.7945
- Precision = 0.9531
- Recall = 0.6812

**UNSW-NB15 + enhanced_mlp_ae**
- ROC-AUC = 0.8872
- PR-AUC = 0.8125
- F1 Score = 0.7638
- Precision = 0.9217
- Recall = 0.6543

### 4.4 结果分析
- 模型在真实数据集上表现良好，能够有效检测网络异常
- 精确率较高，误报率较低，适合实际部署
- 召回率有提升空间，可通过调整阈值或模型结构进一步优化
