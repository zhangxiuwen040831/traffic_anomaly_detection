# 网络流量异常检测系统 - 答辩材料

## 1. 答辩报告

### 1.1 项目概述

本项目实现了一个基于深度学习的网络流量异常检测系统，采用 Autoencoder 架构进行无监督异常检测。系统能够自动识别偏离正常行为模式的网络流量，为网络安全提供新的保障手段。

### 1.2 系统架构

系统采用了五层架构设计：
- **数据层**：数据采集、预处理、特征工程
- **模型层**：Autoencoder 系列模型，混合异常评分
- **评估层**：多维度指标评估，阈值优化
- **部署层**：模型打包，推理服务
- **前端层**：Streamlit 交互式界面

### 1.3 核心技术

- **增强型 MLP Autoencoder**：添加批归一化和 Dropout，提高模型泛化能力
- **混合异常评分方法**：结合重构误差、潜在空间距离、密度估计三种评分方法
- **多阈值策略**：支持 f1_optimal、youden、pr_optimal 三种阈值选择策略

### 1.4 实验结果

**CIC-IDS2017 + enhanced_mlp_ae**
- ROC-AUC = 0.9135
- PR-AUC = 0.8495
- F1 Score = 0.7945
- Precision = 0.9531
- Recall = 0.6812

### 1.5 系统实现

- **后端**：模块化设计，支持多种模型和数据集
- **前端**：Streamlit 交互式界面，支持数据上传和实时推理
- **部署**：完整的 deploy_bundle 结构，便于部署和使用

### 1.6 创新点

- **混合异常评分**：结合多种评分方法，提高检测准确性
- **多阈值策略**：提供多种阈值选择方法，适应不同场景需求
- **端到端系统**：从数据处理到模型部署的完整流程
- **用户友好界面**：直观的前端展示和交互

### 1.7 局限性与未来工作

**局限性**：
- 召回率有待提高
- 模型复杂度较高
- 对数据质量依赖较大

**未来工作**：
- 模型优化与轻量化
- 特征工程改进
- 实时检测支持
- 多源数据融合
- 自适应学习

## 4. 问答题库

### 4.1 为什么选这个数据集？

**回答**：我们选择了CIC-IDS2017和UNSW-NB15这两个数据集，主要基于以下考虑：

- **CIC-IDS2017**：这是一个真实的网络流量数据集，包含了多种现代网络攻击类型，如DoS、DDoS、暴力破解等，更接近真实网络环境
- **UNSW-NB15**：这是网络安全研究中常用的基准数据集，便于与其他研究进行对比
- **数据多样性**：两个数据集的特征和攻击类型有所不同，有助于验证模型的泛化能力
- **公开可用**：这两个数据集都是公开的，便于重复实验和验证

### 4.2 为什么用 Autoencoder / VAE / Hybrid？

**回答**：我们选择Autoencoder系列模型的原因：

- **无监督学习**：Autoencoder可以仅使用正常样本进行训练，不需要标注的异常样本，这在网络安全领域非常重要，因为异常样本往往难以获取
- **特征学习能力**：Autoencoder能够自动学习数据的潜在表示，捕获正常网络流量的特征模式
- **灵活性**：不同类型的Autoencoder（如VAE、增强型MLP Autoencoder）可以适应不同的场景需求
- **计算效率**：相比其他深度学习模型，Autoencoder的计算复杂度相对较低，适合部署到资源受限的环境

### 4.3 为什么训练只用正常样本？

**回答**：训练只用正常样本主要基于以下原因：

- **数据获取难度**：异常样本在实际网络环境中相对稀少，且标注成本高
- **无监督学习优势**：Autoencoder作为无监督学习模型，可以通过学习正常样本的分布来检测异常
- **泛化能力**：仅使用正常样本训练可以使模型更好地学习正常行为模式，从而更准确地识别偏离这些模式的异常
- **适应性**：这种方法可以适应不断变化的网络环境，因为它不需要重新标注新的异常样本

### 4.4 Precision 和 Recall 为什么会权衡？

**回答**：Precision（精确率）和Recall（召回率）之间存在天然的权衡关系：

- **Precision**：衡量模型预测为异常的样本中实际为异常的比例，关注的是减少误报
- **Recall**：衡量实际为异常的样本中被模型正确预测的比例，关注的是减少漏报

当我们调整模型的阈值时，这两个指标会相互影响：
- 提高阈值：会减少误报，提高Precision，但可能会增加漏报，降低Recall
- 降低阈值：会减少漏报，提高Recall，但可能会增加误报，降低Precision

在实际应用中，需要根据具体场景的需求来选择合适的阈值，例如：
- 对误报敏感的场景（如金融系统）：优先考虑高Precision
- 对漏报敏感的场景（如入侵检测）：优先考虑高Recall

### 4.5 你怎么证明没有数据泄漏？

**回答**：我们采取了以下措施来防止数据泄漏：

- **严格的数据划分**：将数据集划分为训练集、验证集和测试集，确保三个集合完全独立
- **正确的预处理流程**：只在训练集上进行特征标准化和编码的拟合，然后应用到验证集和测试集
- **分层采样**：使用分层采样确保各个集合的类别分布一致
- **无交叉验证**：测试集只用于最终评估，不参与任何模型选择或参数调优
- **代码审查**：通过代码审查确保没有在模型训练过程中使用测试集数据

从我们的实验结果来看，模型在测试集上的性能与验证集上的性能相近，这也验证了没有数据泄漏的问题。
