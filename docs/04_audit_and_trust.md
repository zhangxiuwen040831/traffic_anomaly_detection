# 网络流量异常检测系统 - 审计与可信度

## 1. 审计背景

第四阶段报告了接近完美的性能指标：
- ROC-AUC = 1.0000
- PR-AUC = 1.0000
- F1 Score = 0.9983
- Precision = 1.0000
- Recall = 0.9965

这些指标与第三阶段的可信结果（ROC-AUC=0.9135, PR-AUC=0.8495, F1=0.7945）存在显著差异，需要进行独立审计。

## 2. 审计发现

### 2.1 数据来源问题
- **使用合成数据**：第四阶段的CIC-IDS2017数据来自 `synthetic_fallback`，而非真实数据集
- **样本量过小**：测试集仅包含820个样本
- **特征不真实**：使用随机生成的 `feature_0` 到 `feature_35`，与真实网络流量特征无关

### 2.2 数据划分分析
- **划分逻辑正确**：代码中实现了 train/val/test 三集合划分
- **预处理流程正确**：只在训练集上 fit 标准化，然后应用到 val/test
- **无数据泄漏**：三集合之间无重叠

### 2.3 阈值优化分析
- **阈值计算正确**：f1_optimal/youden/pr_optimal 均在验证集上计算
- **阈值应用正确**：计算得到的阈值应用到独立测试集

### 2.4 指标计算分析
- **计算方法正确**：使用标准的 sklearn 指标计算方法
- **结果不可信**：在合成数据上取得接近完美的指标没有实际意义

### 2.5 部署与前端分析
- **前端使用真实数据**：前端应用加载的是 `data/processed/` 目录下的真实数据
- **模型与数据不匹配**：deploy_bundle 中的模型是在合成数据上训练的，与前端使用的数据不一致

## 3. 具体问题根因

1. **数据替换**：CIC-IDS2017 真实数据缺失，系统自动回退到合成数据
2. **样本量过小**：smoke_mode=True 限制了样本数量
3. **合成数据设计**：合成数据的异常样本与正常样本特征差异明显，容易分类
4. **模型-数据不匹配**：部署的模型与前端使用的数据来源不同

## 4. 审计结论

### 4.1 评级：C级（不可信）

**判定理由**：
- 主要使用合成数据进行训练和评估
- 样本量过小，无法代表真实泛化能力
- 接近100%的指标是在易于分类的合成数据上获得的
- 部署的模型与前端使用的数据不匹配

### 4.2 影响范围
- 第四阶段的性能指标不能作为正式结论
- 部署包中的模型在真实数据上可能表现不佳
- 前端展示的结果与模型实际能力不符
