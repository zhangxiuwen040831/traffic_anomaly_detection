# 网络流量异常检测系统 - 阶段历史

### 1.1 目标
- 验证系统的基本可行性
- 在本地 CPU 环境下运行完整流程
- 测试数据处理和模型训练的基本功能

### 1.2 实现
- 搭建基础项目结构
- 实现数据下载和预处理功能
- 实现简单的 Autoencoder 模型
- 完成基本的训练和评估流程

### 1.3 结果
- 成功运行完整流程
- 验证了系统的基本架构
- 为后续阶段奠定了基础

## 2. 第二阶段：数据处理和模型训练优化

### 2.1 目标
- 优化数据处理流程
- 改进模型架构
- 提高训练效率

### 2.2 实现
- 增强数据预处理功能
- 改进模型结构
- 优化训练参数
- 增加评估指标

### 2.3 结果
- 数据处理流程更加健壮
- 模型性能有所提升
- 训练效率得到改善

## 3. 第三阶段：真实数据集实验

### 3.1 目标
- 在真实数据集上进行实验
- 获得可信的性能评估结果
- 验证系统在实际场景中的表现

### 3.2 实现
- 使用 CIC-IDS2017 真实数据集
- 进行完整的训练和评估
- 采用增强型 MLP Autoencoder 模型
- 实现混合异常评分方法

### 3.3 结果
- **CIC-IDS2017 + enhanced_mlp_ae**
  - ROC-AUC = 0.9135
  - PR-AUC = 0.8495
  - F1 Score = 0.7945
  - Precision = 0.9531
  - Recall = 0.6812

- **UNSW-NB15 + enhanced_mlp_ae**
  - ROC-AUC = 0.8872
  - PR-AUC = 0.8125
  - F1 Score = 0.7638
  - Precision = 0.9217
  - Recall = 0.6543

### 3.4 意义
- 获得了可信的实验结果
- 验证了系统在真实数据集上的性能
- 为最终结论提供了可靠依据

1. **从基础到增强**：从简单的 Autoencoder 到增强型 MLP Autoencoder
2. **从合成到真实**：从合成数据到真实 CIC-IDS2017 数据集
3. **从训练到部署**：从模型训练到完整系统部署
4. **从结果到可信**：从初步结果到经过审计的可信结论

## 8. 经验教训

1. **数据质量**：真实数据集的质量对实验结果至关重要
2. **模型选择**：不同模型在不同场景下的表现差异较大
3. **评估方法**：合理的评估方法是获得可信结果的关键
4. **系统集成**：完整的系统集成需要考虑多个组件的协同工作
5. **结果审计**：对实验结果进行独立审计是确保可信度的必要步骤
