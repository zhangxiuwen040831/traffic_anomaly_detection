# 网络流量异常检测系统报告

## 1. 算法架构

### 1.1 模型类型
- **模型名称**: Enhanced MLP Autoencoder
- **模型类型**: 无监督异常检测
- **核心原理**: 自编码器通过学习正常数据的特征表示，对异常数据产生较高的重构误差，从而实现异常检测

### 1.2 网络结构
```
EnhancedMLPAutoencoder(
  (encoder): Sequential(
    (0): Linear(in_features=42, out_features=128, bias=True)
    (1): BatchNorm1d(128, eps=1e-05, momentum=0.1, affine=True, track_running_stats=True)
    (2): ReLU()
    (3): Dropout(p=0.3, inplace=False)
    (4): Linear(in_features=128, out_features=64, bias=True)
    (5): BatchNorm1d(64, eps=1e-05, momentum=0.1, affine=True, track_running_stats=True)
    (6): ReLU()
    (7): Dropout(p=0.3, inplace=False)
    (8): Linear(in_features=64, out_features=32, bias=True)
    (9): BatchNorm1d(32, eps=1e-05, momentum=0.1, affine=True, track_running_stats=True)
    (10): ReLU()
    (11): Dropout(p=0.3, inplace=False)
  )
  (decoder): Sequential(
    (0): Linear(in_features=32, out_features=64, bias=True)
    (1): BatchNorm1d(64, eps=1e-05, momentum=0.1, affine=True, track_running_stats=True)
    (2): ReLU()
    (3): Dropout(p=0.3, inplace=False)
    (4): Linear(in_features=64, out_features=128, bias=True)
    (5): BatchNorm1d(128, eps=1e-05, momentum=0.1, affine=True, track_running_stats=True)
    (6): ReLU()
    (7): Dropout(p=0.3, inplace=False)
    (8): Linear(in_features=128, out_features=42, bias=True)
  )
)
```

### 1.3 关键组件
- **编码器**: 将输入特征压缩为低维表示
- **解码器**: 将低维表示重构为原始输入
- **BatchNorm**: 加速训练收敛，提高模型稳定性
- **Dropout**: 防止过拟合
- **ReLU激活函数**: 引入非线性，增强模型表达能力

### 1.4 模型参数量
- **总参数量**: 32,522

## 2. 数据集内容

### 2.1 数据集基本信息
- **数据集名称**: UNSW-NB15
- **数据集类型**: 网络流量异常检测数据集
- **总样本数**: 175,341
- **特征维度**: 42
- **标签分布**:
  - 正常样本: 56,000 (32%)
  - 异常样本: 119,341 (68%)

### 2.2 数据分割
- **训练集**: 140,272 样本 (80%)
  - 正常样本: 44,800
  - 异常样本: 95,472
- **测试集**: 35,069 样本 (20%)
  - 正常样本: 11,200
  - 异常样本: 23,869

### 2.3 特征类型
- **数值特征**: 包括流量持续时间、数据包数量、字节数、速率等
- **分类特征**: 包括协议类型、服务类型、连接状态等

### 2.4 数据预处理
- **缺失值处理**: 移除包含NaN值的样本
- **分类特征编码**: 使用LabelEncoder将分类特征转换为数值
- **特征标准化**: 使用StandardScaler对特征进行标准化

## 5. 结论与建议

### 5.1 结论
- **模型效果**: 模型在真实网络流量数据上表现良好，异常检测能力明显优于随机水平
- **训练效率**: 训练过程稳定，早停机制有效防止过拟合
- **资源使用**: GPU显存占用低，训练时间合理

### 5.2 建议
1. **模型优化**:
   - 尝试不同的网络架构，如增加隐藏层维度或使用更复杂的网络结构
   - 调整超参数，如学习率、批处理大小、dropout率等
   - 考虑使用其他损失函数，如重构误差与异常分数的组合

2. **数据处理**:
   - 进一步分析特征重要性，选择更有信息量的特征
   - 考虑数据增强，生成更多的正常样本
   - 尝试不同的特征工程方法，如特征选择、特征组合等

3. **评估改进**:
   - 使用更多的评估指标，如精确率-召回率曲线、F1-score曲线等
   - 进行交叉验证，确保模型的泛化能力
   - 与其他异常检测算法进行比较，如隔离森林、LOF等

## 6. 总结

本项目成功实现了基于深度学习的网络流量异常检测系统，使用Enhanced MLP Autoencoder模型对UNSW-NB15数据集进行了训练和评估。模型在1000轮次训练中，通过早停机制在第170轮达到最佳性能，ROC-AUC为0.7868，PR-AUC为0.8635，表现出较好的异常检测能力。

该系统可以作为网络流量异常检测的基础框架，通过进一步优化和扩展，可以应用于实际的网络安全监控场景。
